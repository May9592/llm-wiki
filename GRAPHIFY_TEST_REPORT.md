## 🧪 测试用例

### 测试1: 查询代码工作流程

#### 测试目标
验证 Graphify 能否准确回答关于代码架构和工作流程的复杂问题。

#### 测试问题
**"KnowledgeCompiler 如何协调各个模块工作？"**

#### 执行过程
```bash
# 查询命令
/graphify query "KnowledgeCompiler 如何协调各个模块工作？"

# 执行方式：BFS 遍历，深度 2 层
```

#### 测试结果
**✅ 测试通过**

**查询结果**:
- **发现节点**: 31 个相关节点
- **发现关系**: 33 条连接关系
- **查询深度**: 2 层 BFS 遍历

**核心发现**:
1. **主要协调节点**:
   - `._save_manifest()` - 元数据管理
   - `main()` - 程序入口
   - `.compile()` - 编译协调
   - `.generate()` - 内容生成

2. **设计意图识别**:
   - `# HACK: 简单实现 - 取第一段或第一个标题`
   - 系统正确识别了代码中的技术债务标记

3. **模块关系**:
   - 成功识别了 KnowledgeCompiler 与各模块的调用关系
   - 发现了隐式的协调机制

#### 技术验证
- ✅ **图谱查询**: 成功在 45 个节点的图谱中找到相关内容
- ✅ **关系提取**: 准确提取了函数调用和包含关系
- ✅ **上下文理解**: 正确理解了"协调"和"模块"的语义
- ✅ **响应速度**: <1 秒完成复杂查询

#### 业务价值
- **快速理解**: 新开发者可以快速理解系统架构
- **知识发现**: 发现了代码组织和设计的模式

### 测试2: 路径关系发现

#### 测试目标
验证 Graphify 能否找到两个概念之间的最短连接路径，揭示隐藏的关系。

#### 测试场景
**查找 DocumentReader 到 SummaryGenerator 的连接路径**

#### 执行过程
```bash
# 路径查找命令
/graphify path "DocumentReader" "SummaryGenerator"

# 执行方式：最短路径算法
```

#### 测试结果
**✅ 测试通过**

**路径结果**:
- **最短路径**: 2 跳
- **路径类型**: 结构包含关系

**详细路径**:
```
DocumentReader
    ↓ contains
knowledge_compiler.py
    ↓ contains  
SummaryGenerator
```

#### 路径分析
1. **第一跳**: DocumentReader → knowledge_compiler.py
   - **关系类型**: `contains` (文件包含关系)
   - **关系置信度**: EXTRACTED (从 AST 提取)
   - **业务含义**: DocumentReader 类定义在 knowledge_compiler.py 文件中

2. **第二跳**: knowledge_compiler.py → SummaryGenerator
   - **关系类型**: `contains` (文件包含关系)
   - **关系置信度**: EXTRACTED (从 AST 提取)
   - **业务含义**: SummaryGenerator 类也定义在同一文件中

#### 技术验证
- ✅ **路径查找**: 成功找到最短路径
- ✅ **关系类型**: 正确识别文件包含关系
- ✅ **置信度标记**: 准确标记为 EXTRACTED
- ✅ **业务洞察**: 揭示了两个模块的文件级别关联

#### 业务价值
- **架构理解**: 明确了模块间的物理位置关系
- **依赖分析**: 了解了代码组织的实际结构
- **重构指导**: 为代码重组提供了结构洞察
