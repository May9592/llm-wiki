## 🔍 使用智能查询命令

### 方法1：基础查询

**命令格式**：
```
/graphify query "你的问题"
```

**实用例子**：

#### 例子1：了解工作流程
```
/graphify query "知识库如何编译文档？"
```

**结果**：告诉你整个编译过程，从读取文档到生成摘要

#### 例子2：查找设计原则
```
/graphify query "代码设计原则是什么？"
```

**结果**：从代码注释中提取设计原则

#### 例子3：理解错误处理
```
/graphify query "错误处理是怎么做的？"
```

**结果**：找到相关的错误处理代码

---

### 方法2：查找关系

**命令格式**：
```
/graphify path "功能A" "功能B"
```

**实用例子**：

#### 例子1：查找数据流向
```
/graphify path "DocumentReader" "SummaryGenerator"
```

**结果**：展示从读取文档到生成摘要的路径

**你会看到**：
```
DocumentReader (读取文档)
    ↓ contains
knowledge_compiler.py (在同一个文件中)
    ↓ contains  
SummaryGenerator (生成摘要)
```

#### 例子2：查找调用关系
```
/graphify path "main()" "KnowledgeCompiler"
```

**结果**：展示程序入口如何调用主编译器

---

### 方法3：详细解释某个功能

**命令格式**：
```
/graphify explain "功能名称"
```

**实用例子**：

#### 例子1：理解核心类

**结果**：

- 错误处理机制

## 🎯 小白专属技巧

### 技巧1：从简单问题开始

**推荐的问题类型**：
- ✅ "这个类是做什么的？"
- ✅ "X和Y有什么关系？"
- ✅ "数据是如何从A流向B的？"

**避免的问题类型**：
- ❌ 技术细节太深的问题
- ❌ 多个复杂概念混合的问题
- ❌ 需要大量背景知识的问题

### 技巧2：使用自然语言

**好的提问方式**：
```bash
# ✅ 自然、直接
/graphify query "数据是怎么清洗的？"

# ✅ 日常用语
/graphify query "为什么要有摘要生成器？"

# ✅ 关心业务
/graphify query "用户数据是如何处理的？"
```

**避免的提问方式**：
```bash
# ❌ 技术术语过多
/graphify query "AST extraction algorithm implementation"

# ❌ 问题太宽泛
/graphify query "代码怎么工作？"

# ❌ 没有具体目标
/graphify query "告诉我一些信息"
```

### 技巧3：多问几次

**学习策略**：
1. 先问整体：`"主要功能有哪些？"`
2. 再问细节：`"数据读取是怎么做的？"`
3. 最后问关系：`"读取和生成如何配合？"`

**好处**：
- 从宏观到微观，逐步深入
- 建立完整的知识体系
- 不会一次性被信息淹没
