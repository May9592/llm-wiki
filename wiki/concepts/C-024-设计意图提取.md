# C-024: 设计意图提取

## 定义
自动从代码和文档中提取"为什么这样设计"的信息，将隐性的设计决策显性化的技术。

## 核心特征
- **多源提取**：从 docstring、行内注释、设计文档中提取
- **结构化存储**：作为 `rationale_for` 节点存储在知识图谱中
- **上下文保留**：保留设计决策的完整语境

## 提取方式

### 1. 代码注释标记
自动识别特殊注释标记：
- **NOTE**：一般性说明
- **IMPORTANT**：重要设计原则
- **HACK**：临时解决方案或已知问题
- **TODO**：计划改进的地方
- **WHY**：设计决策的原因

### 2. 文档分析
- 从文档中提取设计动机
- 识别权衡和取舍
- 记录架构决策

### 3. 推断关系
- 基于代码结构推断隐式设计意图
- 分析调用关系背后的设计模式
- 识别架构风格

## 在 Graphify 中的实现

### 节点类型
```python
rationale_for 节点：
{
  "id": "main_rationale_1",
  "label": "Knowledge Base Compiler 编译知识库的核心模块",
  "source_location": "raw/code/knowledge_compiler.py:8-15",
  "confidence": "EXTRACTED"
}
```

### 关系类型
- **rationale_for**：从解释指向概念节点
- **describes**：文档描述指向设计概念

## 实际价值

### 对代码理解
- **知道"为什么"**：不仅理解代码做了什么，还知道为什么这样设计
- **设计决策追踪**：了解历史设计决策
- **技术债务识别**：通过 HACK 和 TODO 标记发现问题

### 对团队协作
- **知识传承**：设计意图不会因人员流动而丢失
- **决策透明**：设计原则和权衡变得明确
- **重构指导**：了解原始设计意图，避免破坏性修改
